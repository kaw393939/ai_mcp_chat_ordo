# Chapter 8 — Organizational Accessibility Maturity

## Abstract
Accessibility is not a team — it is an organizational capability. This chapter covers the maturity model from ad hoc ("we fix issues when someone complains") to systematic ("accessibility is part of our definition of done").

---

## Maturity Levels

| Level | Description |
|-------|-------------|
| 1. Ad hoc | Fix issues only when reported or legally required |
| 2. Planned | Accessibility goals exist but are not consistently measured |
| 3. Managed | Standards defined, training provided, testing regular |
| 4. Measured | KPIs track accessibility quality, regressions caught in CI |
| 5. Optimized | Inclusive design integrated into all processes, continuous improvement |

## Chapter Checklist
- What maturity level is your organization at today?
- Is there a named accessibility champion or team?
- Does your definition of "done" include accessibility criteria?
